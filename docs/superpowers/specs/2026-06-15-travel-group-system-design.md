# 旅游团火车票记录系统设计文档

## 一、项目概述

旅行社内部火车票管理系统，用于记录旅游团的票务信息、人员明细和费用计算。

## 二、技术栈

| 层级 | 技术 |
|------|------|
| 前端 | Vue 3 + Element Plus |
| 后端 | Node.js + Express |
| 数据库 | SQLite |
| ORM | Prisma |

## 三、数据模型

### 3.1 旅游团表

| 字段 | 说明 |
|------|------|
| 编号 | 主键 |
| 团名 | 旅游团名称 |
| 出发日期 | 行程出发日期 |
| 返回日期 | 行程返回日期 |
| 车次 | 火车车次号 |
| 行程 | 发站-到站（如：北京-上海） |
| 成人票价 | 成人票单价 |
| 儿童票价 | 儿童票单价 |
| 核验数 | 外国证件核验次数（手动输入） |
| 电子发票数 | 电子发票数量（手动输入） |

| 字段 | 说明 |

| 编号 | 主键 |

| 证件类型 | 二代/外护 |

### 3.3 费用配置表

| 字段 | 说明 | 默认值 |
|------|------|--------|
| 编号 | 主键 | |
| 购票服务费 | 每张票的服务费 | 10元 |
| 退票服务费 | 每张退票的服务费 | 20元 |
| 外国证件核验费 | 每次核验费用 | 8元 |
| 电子发票费 | 每张电子发票费用 | 3元 |

## 四、票型判断逻辑

导入人员时自动判断票型：

- **中国身份证（二代）**：根据证件号码计算年龄，6周岁 ≤ 年龄 < 14周岁 为儿童，否则为成人
- **外国护照（外护）**：根据票价判断，票价 ≤ 成人票价×50% 为儿童，否则为成人

## 五、费用计算规则

| 项目 | 计算方式 |
|------|---------|
| 票价小计 | 成人票数×成人票价 + 儿童票数×儿童票价 |
| 购票服务费 | 有效购票数 × 购票服务费标准 |
| 退票服务费 | 退票数 × 退票服务费标准 |
| 核验费 | 核验数 × 核验费标准 |
| 电子发票费 | 电子发票数 × 电子发票费标准 |
| 共计 | 以上各项求和 |

**退票规则**：退票人员的购票服务费不退还，且需额外收取退票服务费。

## 六、页面结构

### 6.1 团管理页
- 显示所有旅游团列表
- 支持新增、编辑、删除团
- 点击进入团详情

### 6.2 团详情页
- 团汇总信息区（动态计算的费用明细）
- 人员明细列表（可批量导入、添加、编辑、删除、标记退票）
- 复制团信息按钮

### 6.3 费用配置页
- 查看和修改费用标准

## 八、批量导入格式

导入的 Excel 格式（来自票务系统导出）：

| 序号 | 证件 | 姓名 | 证件号码 | 日期 | 车次 | 发站 | 到站 | 席别 | 车厢 | 座号 | 票价 | 订单号 |
|------|------|------|---------|------|------|------|------|------|------|------|------|--------|

- 证件类型：二代 = 中国身份证，外护 = 外国护照
- 状态：导入时默认为"正常"
- 是否开发票：导入后手动勾选

## 十、API 接口

| 功能 | 方法 | 接口 |
|------|------|------|
| 获取旅游团列表 | GET | /api/groups |
| 新增旅游团 | POST | /api/groups |
| 编辑旅游团 | PUT | /api/groups/:id |
| 删除旅游团 | DELETE | /api/groups/:id |
| 获取团详情 | GET | /api/groups/:id |
| 复制团信息 | GET | /api/groups/:id/copy |
| 获取人员列表 | GET | /api/groups/:id/members |

| 人员退票 | PUT | /api/members/:id/refund |

| 修改费用配置 | PUT | /api/config |
